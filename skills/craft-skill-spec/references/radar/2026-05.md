## What changed in the landscape

Recent model guidance is not only about provider-specific settings. It strengthens a cross-agent pattern:

- modern models follow scoped instructions more tightly, so contradictions and unclear priority order cost more
- agentic prompts work better with explicit completion and verification checks than with vague "be thorough" pressure
- missing-context behavior should be stated directly: retrieve, ask, or proceed with labeled assumptions
- newer agents are more proactive with tools and subagents, so old anti-laziness prompts can overcorrect into wasteful action
- source verification and success criteria are now first-class guidance for research workflows

## Candidate patterns

### Pattern: explicit verification gates

- observed signal: OpenAI recommends checking requirements, grounding, format, and side effects before finalizing high-impact agent work; Anthropic emphasizes success criteria and source verification for research.
- source support: OpenAI GPT-5.5 prompt guidance; Anthropic Claude prompting best practices.
- likely benefit: Catches partial completion, unsupported claims, format drift, and unsafe irreversible actions.
- likely risk: Can become ceremony on tiny one-shot prompts.
- proposed status: `adopt`
- CraftKit implication: Add verification checks to prompt quality review and system-prompt templates, while keeping simple prompts short.

### Pattern: missing-context gating

- observed signal: Latest guidance favors explicit rules for when the agent should look up missing context, ask a minimal question, or proceed with assumptions.
- source support: OpenAI GPT-5.5 prompt guidance; Anthropic research guidance.
- likely benefit: Reduces hallucinated facts and unnecessary clarification loops.
- likely risk: Overly rigid gates can stop useful work when assumptions would be harmless.
- proposed status: `adopt`
- CraftKit implication: Make missing-context handling part of prompt sharpening and failure-mode diagnosis.

### Pattern: instruction hierarchy review

- observed signal: OpenAI highlights contradictory prompts as especially damaging for tightly instruction-following models.
- source support: OpenAI GPT-5.5 prompt guidance.
- likely benefit: Improves reliability without adding length.
- likely risk: None if framed as a review check rather than a new section every prompt must carry.
- proposed status: `adopt`
- CraftKit implication: Add conflict checks to `craft-prompt` and an explicit failure mode to `craft-tune`.

### Pattern: calibrated autonomy instead of anti-laziness

- observed signal: Provider guidance now warns that models can be proactive enough to overuse tools or subagents when old persistence prompts are left in place.
- source support: OpenAI GPT-5.5 prompt guidance; Anthropic Claude prompting best practices.
- likely benefit: Preserves useful autonomy while reducing wasted tool calls, spurious subagents, and file churn.
- likely risk: Underprompting can still cause premature stopping on low-effort or constrained models.
- proposed status: `adopt`
- CraftKit implication: Prefer criteria-driven tool/subagent rules over blanket "keep going" or "use tools aggressively" language.

### Pattern: volatile model catalogs inside reusable guides

- observed signal: Provider model, price, and benchmark facts now change faster than durable prompting principles.
- source support: Current provider docs and CraftKit's existing time-sensitive core-spine warning.
- likely benefit of rejecting it: Lower maintenance burden and fewer stale claims.
- likely risk if copied: Users trust an obsolete guide and choose settings or models incorrectly.
- proposed status: `avoid`
- CraftKit implication: Keep provider guides focused on durable prompting behavior and link to current docs for model selection.

## Adopt this cycle

- explicit verification gates
- missing-context gating
- instruction hierarchy review
- calibrated autonomy instead of anti-laziness

## Avoid this cycle

- volatile model catalogs inside reusable guides
- broad "be thorough / use tools aggressively / never stop" prompts without task-specific completion criteria

## Watch, not default

- provider-specific reasoning and verbosity parameters — useful in deployment notes, not portable prompt cores
- subagent orchestration guidance — useful when tools exist, but still host-specific enough to isolate
