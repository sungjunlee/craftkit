# Gemini

- last reviewed: `2026-05-01`

- **XML tags or Markdown headings** — both work as delimiters. Pick one and stay consistent within a prompt. Google officially shows `<context>`, `<task>`, `<role>`, `<constraints>` examples
- **Use host settings for reasoning depth** when available. Keep prompt text focused on the task, constraints, and desired output rather than asking the model to narrate its thinking.
- **Treat Gemini as an execution engine** — direct, efficient prompts work best. Ask for more detail only where the output needs it.
- **Strong multimodal fit** — when using media inputs, place media first, then ask specific questions about it.
- **Prefers few-shot over zero-shot** — provide examples when possible for better results
- **Verify model and image-generation settings in current Google docs** before recommending model names, prices, context windows, or image tiers.
- When delivering: suggest higher reasoning settings for complex analysis only when the target interface exposes them.
